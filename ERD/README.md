# Описание значимости артефакта (схемы баз данных)

---

| Раздел                           | Содержание                                                                                                                                                                                                                                                                                                                          |
| -------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Процесс и контекст использования | ER-модель создаётся на этапе проектирования системы после согласования бизнес-требований и основных бизнес-процессов. Используется при проектировании сервиса Decision Orchestrator, определении структуры хранения данных в PostgreSQL и MongoDB, а также при проектировании API и бизнес-логики обработки кредитных заявок.       |
| Цель создания                    | Зафиксировать структуру данных, необходимых для обработки кредитной заявки, хранения результатов принятия решения, кредитных политик, результатов выполнения правил и аудита. Обеспечить единое понимание модели данных между аналитиком, архитектором и разработчиками до начала реализации.                                       |
| Что становится определено        | Определён состав бизнес-сущностей (Decision Request, Applicant, Decision, Credit Policy и др.), их атрибуты, связи между ними, распределение данных между PostgreSQL и MongoDB, а также правила хранения информации, необходимой для воспроизводимости решений и аудита.                                                            |
| Пользователи артефакта           | Системный аналитик использует модель для детализации требований и проектирования API. Backend-разработчики — при реализации схем БД, ORM-моделей и запросов. QA-инженеры — для подготовки тестовых данных и проверки корректности сохранения информации.                                                                            |
| Использование в дальнейшем       | На основе ER-модели создаются SQL-скрипты PostgreSQL, коллекции MongoDB, JSON Schema, ORM-модели, API-контракты, а также реализуется бизнес-логика обработки кредитной заявки и хранения результатов выполнения правил. Модель используется при проектировании интеграций и подготовке тестовых сценариев.                          |
| Последствия отсутствия           | Без ER-модели разработчики могут по разному трактовать структуру хранения данных, что приведёт к несовместимым моделям PostgreSQL и MongoDB, дублированию данных, ошибкам при реализации связей между сущностями, невозможности воспроизвести принятое кредитное решение и усложнению интеграции компонентов Decision Orchestrator. |
