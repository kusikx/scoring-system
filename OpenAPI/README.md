# Описание значимости артефакта (OpenAPI спецификация)

---

| **Раздел**                           | **Содержание**                                                                                                                                                                                                                                                                                                                                                                           |
| ------------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Процесс и контекст использования** | Артефакт используется на этапе проектирования интеграционного взаимодействия между кредитором и скоринговой платформой. OpenAPI-спецификация применяется после определения бизнес-процесса обработки заявки и модели данных, когда необходимо формализовать REST API Decision Orchestrator и зафиксировать формат обмена сообщениями между системой и внешними клиентами.                |
| **Цель создания**                    | Зафиксировать единый контракт взаимодействия между кредиторами и Decision Orchestrator, определить доступные REST-методы, структуру запросов и ответов, форматы ошибок.                                                                                                                                                                                                                  |
| **Что становится определено**        | Определены конечные точки API, HTTP-методы, схемы запросов и ответов, обязательные заголовки, коды ответов, а также правила взаимодействия клиента с системой при создании и получении результатов обработки кредитной заявки.                                                                                                                                                           |
| **Пользователи артефакта**           | Системный аналитик использует спецификацию для детализации требований к интеграциям. Backend-разработчики реализуют REST API согласно контракту. Команда кредитора разрабатывает интеграцию с платформой, используя OpenAPI как техническую документацию. QA-инженеры формируют тестовые сценарии и проверяют соответствие реализации утвержденному контракту.                           |
| **Использование в дальнейшем**       | На основе OpenAPI реализуются REST-эндпоинты Decision Orchestrator, разрабатываются интеграции кредиторов с платформой, а также подготавливаются тесты. Спецификация используется как единый источник истины при сопровождении и развитии API.                                                                                                                                           |
| **Последствия отсутствия**           | Без OpenAPI-спецификации участники проекта могут по-разному трактовать формат запросов, ответов и правила взаимодействия с Decision Orchestrator. Это приведет к несовместимым реализациям API, ошибкам интеграции между кредиторами и платформой, увеличению количества доработок при подключении новых клиентов и усложнит автоматизацию тестирования и сопровождение REST-контрактов. |
