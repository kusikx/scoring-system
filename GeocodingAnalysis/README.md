# Сравнительный анализ сервисов геокодирования

## Общая информация

**Цель исследования:** выбрать наиболее подходящий сервис для выполнения прямого и обратного геокодирования в скоринговой платформе.

В ходе работы были исследованы API следующих сервисов: Яндекс.Геокодер, DaData, GigaChat.

Исследование выполнялось путем изучения документации сервисов, автоматизированного выполнения запросов при помощи Python-скрипта, анализа полученных результатов.

Для тестирования использовались:

* 100 адресов для прямого геокодирования;
* 100 координат для обратного геокодирования.

Всего было выполнено **600 запросов** (200 на каждый сервис).

## Критерии оценки

| Критерий                                                   | Описание                                                                                                                                |
| ------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| Поддержка прямого геокодирования     | Преобразование почтового адреса в географические координаты                               |
| Поддержка обратного геокодирования | Преобразование географических координат в почтовый адрес                                     |
| Скорость ответа                                      | Среднее время выполнения запроса отдельно для каждого сценария                           |
| Точность результата                              | Доля успешно выполненных запросов и наличие информации о качестве результата |
| Простота интеграции                              | Сложность использования API и требования к аутентификации                                       |
| Назначение сервиса                                | Соответствие сервиса задаче геокодирования                                                               |

## Анализ сервисов

### Яндекс.Геокодер

#### Преимущества

* специализированный сервис геокодирования;
* поддерживает прямое и обратное геокодирование;
* обеспечивает наименьшее среднее время ответа среди протестированных сервисов;
* предоставляет информацию об уровне точности найденного объекта;
* простой REST API и единый механизм работы для обоих сценариев.

#### Недостатки

* не выполняет очистку и нормализацию пользовательских адресов;
* качество обратного геокодирования оказалось ниже остальных протестированных сервисов (65% успешных результатов).

### DaData

#### Преимущества

* выполняет очистку и нормализацию адресов;
* поддерживает прямое и обратное геокодирование;
* предоставляет показатель качества определения координат;
* показала высокую успешность как при прямом, так и при обратном геокодировании (96%).

#### Недостатки

* используются разные ендпоинты для прямого и обратного геокодирования;
* среднее время ответа значительно выше, чем у Яндекс.Геокодера;

### GigaChat

#### Преимущества

* способен определять адреса и координаты по текстовому описанию;
* может обрабатывать запросы со свободной формулировкой;
* не требует специализированного API геокодирования.

#### Недостатки

* не является специализированным сервисом геокодирования;
* отсутствуют показатели качества определения результата;
* результаты могут отличаться при повторных запросах;
* требуется получение OAuth-токена;
* среднее время ответа является наибольшим среди протестированных сервисов.

## Вывод

**Яндекс.Геокодер** продемонстрировал наименьшее среднее время ответа и обеспечил 100% успешность прямого геокодирования. При этом успешность обратного геокодирования составила 65%, что оказалось ниже остальных протестированных сервисов.

**DaData** показала наиболее сбалансированные результаты, обеспечив 96% успешных запросов как при прямом, так и при обратном геокодировании. Дополнительным преимуществом сервиса является возможность очистки и нормализации адресов, а также наличие показателя качества определения координат.

**GigaChat** успешно обработал все тестовые запросы, однако данный показатель отражает успешность выполнения HTTP-запросов, а не объективную точность геокодирования. Поскольку сервис является языковой моделью, результаты могут изменяться при повторных запросах и не сопровождаются специализированными метриками качества.

### Выбор сервиса

**Яндекс.Геокодер** рекомендуется использовать в качестве основного сервиса геокодирования в скоринговой платформе благодаря высокой производительности, специализированному API и высокой точности прямого геокодирования.

**DaData** целесообразно использовать на этапе предварительной обработки пользовательских адресов для их очистки, нормализации и валидации перед передачей в основной геокодер, а также как резервный источник географической информации.

**GigaChat** не рекомендуется использовать в качестве основного механизма геокодирования, поскольку он не является специализированным геоинформационным сервисом и не предоставляет объективных показателей качества полученных результатов.
